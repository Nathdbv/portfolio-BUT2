Contexte & Mission

Dans le cadre de la SAE 3.01, nous avons dû répondre à un besoin client réel : fournir une infrastructure CTF robuste permettant de supporter des dizaines d’utilisateurs simultanés tout en garantissant une étanchéité totale entre les challenges.

L’objectif était de transformer un VPS Debian brut en une plateforme ludique, monitorée et hautement sécurisée via filtrage IPv4 (allow-list).

Mon Rôle Stratégique

  • Lead Infra : Orchestration de l’architecture Docker (compose global, volumes persistants).
  • Sécurité Réseau : Configuration du reverse proxy Nginx avec restriction d’accès par IP.
  • Intégration Plugins : Adaptation du code Python des plugins tiers (Rôles, Multi-flags, QCM).
  • UI & UX : Intégration et personnalisation du thème visuel Case 404.

Écosystème Docker & Sécurité

Chaque défi (Enquêtes, Forensic, Web, etc.) est encapsulé dans son propre container Docker pour éviter toute fuite ou compromission de l'hôte. Portainer nous permet de monitorer l'état des services en temps réel.

  • Orchestration via Docker Compose.
  • Isolation des processus & limitation des ressources.
  • Allow-list IPv4 dynamique sur Nginx.
  • Persistence via volumes Docker sécurisés.

nginx / allow-list.conf

# Filtrage des accès
allow 192.168.x.x; # Jury
allow 10.x.x.x;    # Equipe Dev
deny all;          # Zero Trust
